深入理解 epoll I/O 多路复用

2026-05-16 · Linux · 系统编程 · 阅读时间 12 分钟

Epoll 是 Linux 内核提供的一种高效的 I/O 事件通知机制，是构建高性能网络服务器的基石。

📖 什么是 I/O 多路复用？

I/O 多路复用允许单个线程同时监控多个文件描述符，当其中任何一个就绪时（可读、可写），就会通知应用程序。

🔄 Epoll 工作流程

  1. 创建：使用 epoll_create() 创建 epoll 实例
  2. 注册：使用 epoll_ctl() 添加要监控的文件描述符
  3. 等待：使用 epoll_wait() 等待事件发生
  4. 处理：处理就绪的事件

⚡ LT vs ET 模式

水平触发（LT - Level Triggered）

只要文件描述符处于就绪状态，每次调用 epoll_wait() 都会返回该事件。

边缘触发（ET - Edge Triggered）

只有在状态变化时才通知一次，必须一次性处理完所有数据。

// ET 模式需要循环读取直到 EAGAIN
while (true) {
    ssize_t n = read(fd, buffer, sizeof(buffer));
    if (n < 0 && errno == EAGAIN) {
        break;  // 数据读完
    }
    // 处理数据...
}

🎯 最佳实践

  • 使用 ET 模式 + 非阻塞 I/O 获得最佳性能
  • 合理设置最大事件数，避免内存浪费
  • 及时处理事件，避免事件堆积
  • 注意文件描述符的生命周期管理

💡 总结

Epoll 是 Linux 下构建高并发服务器的首选方案，理解其工作原理对于网络编程至关重要。
